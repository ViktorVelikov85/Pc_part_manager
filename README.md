# Pc_part_manager 💻🔋

Интуитивно конзолно C# приложение за управление на хардуерни компоненти (компютърни части). Проектът използва обектно-ориентиран подход (ООП), архитектурния модел Data Access Layer (DAL) и релационна база данни MariaDB.

---

## ✨ Ключови функционалности

* **Пълен CRUD цикъл:** Прецизно добавяне, извеждане, редактиране и изтриване на компоненти директно от/в базата данни.
* **Търсене и Филтриране:** Бързо локализиране на части по име/производител, както и филтриране по специфична продуктова категория.
* **Полиморфизъм в действие:** Системата автоматично мапва редовете от MariaDB към съответните ООП обекти и използва полиморфизъм за визуализация на уникалните им технически спецификации.
* **UTF-8 кодиране:** Пълна поддръжка на кирилица в конзолния интерфейс, предотвратяваща появата на нечетими символи (`?`).
* **Стабилна валидация:** Защита срещу въвеждане на некоректни бизнес данни (отрицателни цени, празни низове или невалидни количества).

## 📂 Структура на проекта

Архитектурата е изчистена и разделена на два основни логически слоя:

```text
Pc_part_manager/
│
├── 📁 Models/               # Обектно-ориентиран модел данни
│   ├── 📁 PcParts/          # Специфични хардуерни наследници
│   │   ├── Cpu.cs
│   │   ├── Gpu.cs
│   │   ├── Ram.cs
│   │   ├── Motherboard.cs
│   │   └── Ssd.cs
│   ├── Category.cs          # Композиция (Категория на компонента)
│   └── ComputerPart.cs      # Базов абстрактен клас
│
├── 📁 DAL/                  # Data Access Layer (Слой за връзка с БД)
│   ├── DbConfig.cs          # Помощни SQL методи (Execute/GetTable)
│   └── PartRepository.cs    # Пълен бизнес инвентар & CRUD операции
│
└── Program.cs               # Интерактивен потребителски UI (Конзолно меню)
